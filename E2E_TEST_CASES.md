# FastDrop E2E 测试用例

> 由用户在真实设备（Android 手机 + Windows 电脑）上执行。
> 每个用例标注 **前置条件、操作步骤、预期结果**。

---

## 环境准备

| 项目 | 说明 |
|------|------|
| 电脑 | Windows 10/11，运行 `fastdrop.exe`，浏览器打开 `http://127.0.0.1:9527` |
| 手机 | Android，安装最新 FastDrop APK，与电脑在同一局域网 |
| 默认下载目录（PC） | `C:\Users\<用户名>\Downloads\FastDrop` |
| 默认下载目录（手机） | 应用文档目录下的 `FastDrop/` |

---

## 一、配对流程

### TC-1.1 首次扫码配对
**前置：** 电脑刚启动 fastdrop.exe，浏览器显示 QR 码；手机未配对过。
1. 手机打开 FastDrop → 进入扫码页面
2. 扫描电脑上的 QR 码
3. 电脑弹出 "Pairing Request" 对话框，显示手机名称
4. 点击 **Accept**

**预期：**
- [ ] 电脑页面切换到已配对状态（QR 消失，显示拖拽上传区）
- [ ] 手机显示已连接，WebSocket 状态为 connected
- [ ] 电脑 WS 指示灯变绿

### TC-1.2 拒绝配对
**前置：** 同 TC-1.1 步骤 1-3
1. 点击 **Reject**

**预期：**
- [ ] 电脑对话框关闭，继续显示 QR 码
- [ ] 手机提示配对被拒绝或回到扫码页

### TC-1.3 QR 码过期自动刷新
**前置：** 电脑显示 QR 码，不操作
1. 等待 QR 码倒计时归零（约 60 秒）

**预期：**
- [ ] QR 码自动刷新，倒计时重新开始

### TC-1.4 刷新页面后恢复会话（#3）
**前置：** 已完成配对，WS 已连接
1. 在浏览器按 F5 刷新页面

**预期：**
- [ ] 页面刷新后自动恢复到已配对状态（不显示 QR 码）
- [ ] WS 指示灯重新变绿
- [ ] 历史记录仍然可见

### TC-1.5 关闭标签页后重新打开
**前置：** 已完成配对
1. 关闭浏览器标签页
2. 重新打开 `http://127.0.0.1:9527`

**预期：**
- [ ] 显示 QR 码（sessionStorage 随标签关闭而清除，符合预期）
- [ ] 重新扫码可正常配对

## 二、手机 → 电脑 传文件

### TC-2.1 传单张照片
**前置：** 已配对，WS 已连接
1. 手机选择 1 张照片 → 发送
2. 电脑弹出 "Incoming Files" 提示
3. 点击 **Accept**

**预期：**
- [ ] 电脑显示传输进度条，状态从 transferring → completed
- [ ] 文件出现在电脑下载目录 `Downloads\FastDrop\`
- [ ] 历史记录显示 📥 + completed
- [ ] 文件大小与原图一致

### TC-2.2 传多张照片（3张）
**前置：** 已配对
1. 手机选择 3 张照片 → 发送
2. 电脑 Accept

**预期：**
- [ ] 3 个文件都传输完成
- [ ] 下载目录出现 3 个文件

### TC-2.3 传大文件（>100MB，如视频）
**前置：** 已配对
1. 手机选择一个 >100MB 的视频 → 发送
2. 电脑 Accept

**预期：**
- [ ] 传输过程中显示进度和速度
- [ ] 传输完成，SHA-256 校验通过
- [ ] 文件可正常播放

### TC-2.4 传非媒体文件（PDF/ZIP/DOC）
**前置：** 已配对
1. 手机选择一个 PDF 或 ZIP 文件 → 发送
2. 电脑 Accept

**预期：**
- [ ] 文件完整传输，可正常打开

### TC-2.5 同名文件冲突（重命名策略）
**前置：** 已配对，下载目录已有一个 `photo.jpg`
1. 手机再传一个同名 `photo.jpg`
2. 电脑 Accept

**预期：**
- [ ] 新文件保存为 `photo (1).jpg`，原文件不被覆盖

### TC-2.6 传输中暂停/恢复
**前置：** 已配对，正在传一个大文件
1. 传输过程中点击 **Pause**
2. 等待几秒
3. 点击 **Resume**

**预期：**
- [ ] 暂停时进度条停止，状态显示 paused
- [ ] 恢复后继续传输，最终完成

### TC-2.7 传输中取消
**前置：** 已配对，正在传输
1. 点击 **Cancel**

**预期：**
- [ ] 传输项从活跃列表消失
- [ ] 下载目录不产生残留文件（.part 被清理）
- [ ] 历史记录显示 cancelled

### TC-2.8 拒绝接收
**前置：** 已配对
1. 手机发送文件
2. 电脑弹出 Incoming Files 时点击 **Reject**

**预期：**
- [ ] 提示消失，不下载文件
- [ ] 手机侧显示传输被拒绝

## 三、电脑 → 手机 传文件

### TC-3.1 拖拽文件到电脑页面
**前置：** 已配对，WS 已连接
1. 从 Windows 资源管理器拖一个文件到浏览器的拖拽区
2. 手机弹出接收提示

**预期：**
- [ ] 电脑显示 "staged X/Y chunks" → "offered to phone"
- [ ] 手机收到文件传输通知
- [ ] 手机 Accept 后文件下载到手机

### TC-3.2 点击浏览选择文件
**前置：** 已配对
1. 点击拖拽区（或点击浏览按钮）
2. 选择 1-2 个文件

**预期：**
- [ ] 同 TC-3.1，文件成功传到手机

### TC-3.3 传多个文件到手机
**前置：** 已配对
1. 拖入 3 个文件

**预期：**
- [ ] 3 个文件依次 staged 并 offered
- [ ] 手机全部接收成功

## 四、下载路径可配置（#6）

### TC-4.1 PC端修改下载目录
**前置：** 已配对，电脑页面已打开
1. 点击页面右上角 ⚙ 齿轮按钮
2. 设置面板展开，显示当前下载目录
3. 修改为新路径（如 `D:\MyFiles\FastDrop`）
4. 点击 **Save**

**预期：**
- [ ] 设置面板关闭
- [ ] 再次点击齿轮，显示新路径
- [ ] 手机传一个文件 → 文件保存到新目录 `D:\MyFiles\FastDrop\`
- [ ] 旧目录不再收到新文件

### TC-4.2 PC端下载目录重启后生效
**前置：** 已修改下载目录为自定义路径
1. 关闭 fastdrop.exe（Ctrl+C）
2. 重新启动 fastdrop.exe
3. 打开浏览器，配对手机
4. 手机传一个文件

**预期：**
- [ ] 文件保存到之前设置的自定义目录（config.json 已持久化）

### TC-4.3 手机端修改下载目录
**前置：** 已配对
1. 手机进入 Settings 页面
2. 点击 "Download Directory" → "Save location"
3. 输入新路径（如 `/sdcard/Download/MyFastDrop`）
4. 点击 Save

**预期：**
- [ ] 显示 SnackBar 确认
- [ ] 设置页显示新路径
- [ ] 电脑传一个文件到手机 → 文件保存到新路径

### TC-4.4 手机端恢复默认目录
**前置：** 已修改过手机下载目录
1. 进入 Settings → Download Directory
2. 清空路径，点击 Save

**预期：**
- [ ] 显示 "Using default download directory"
- [ ] 后续文件保存到默认位置

## 五、相册重复问题（#1）

### TC-5.1 传照片后相册不重复
**前置：** 已配对
1. 手机从相册选择 1 张照片 → 发送到电脑
2. 传输完成后打开手机相册/图库

**预期：**
- [ ] 相册中该照片只出现一次（不重复）
- [ ] 文件管理器的 file_picker 缓存目录有 `.nomedia` 文件

### TC-5.2 多次传同一照片
**前置：** 已配对
1. 连续发送同一张照片 3 次

**预期：**
- [ ] 相册中仍然只有一张（缓存被清理）

## 六、切换配对（#2 简化版）

### TC-6.1 手机断开配对后重新扫码
**前置：** 手机已配对电脑 A
1. 手机进入 Settings → 点击 **Unpair**
2. 确认对话框 → 点击 Unpair
3. 手机回到扫码页
4. 扫描电脑 A（或另一台电脑 B）的 QR 码
5. 电脑 Accept

**预期：**
- [ ] Unpair 后 WS 断开，session 清除
- [ ] 重新扫码配对成功
- [ ] 传输功能正常

### TC-6.2 电脑重启后重新配对
**前置：** 已配对
1. 电脑关闭 fastdrop.exe
2. 重新启动
3. 浏览器打开页面

**预期：**
- [ ] 显示 QR 码（服务端 session 已失效）
- [ ] 手机重新扫码可配对

## 七、异常场景

### TC-7.1 传输中断网（WiFi 关闭）
**前置：** 正在传一个大文件
1. 手机关闭 WiFi
2. 等待 15 秒
3. 重新打开 WiFi

**预期：**
- [ ] 电脑显示传输失败或设备断开
- [ ] 重连后不崩溃，可重新发起传输

### TC-7.2 手机锁屏后恢复
**前置：** 正在传输
1. 手机按电源键锁屏
2. 等待 10 秒
3. 解锁

**预期：**
- [ ] 传输可能中断（Android 后台限制）
- [ ] 解锁后 app 不崩溃，可重新操作

### TC-7.3 电脑端未配对就拖文件
**前置：** 未配对（显示 QR 码）
1. 拖一个文件到页面

**预期：**
- [ ] 显示提示 "Pair a phone first before sending files."
- [ ] 不崩溃

### TC-7.4 同时从两个浏览器打开
**前置：** 电脑已启动 fastdrop.exe
1. 浏览器 A 打开 `http://127.0.0.1:9527`，完成配对
2. 浏览器 B 打开同一地址

**预期：**
- [ ] 浏览器 B 显示 QR 码（新 session）
- [ ] 浏览器 A 不受影响
