## 能做但需要单独规划 📋

### C. 所有交互在 exe 窗口内，不用浏览器（#4）

**用户诉求：** 双击 exe → 直接出现操作窗口，不打开浏览器。

**为什么不能马上做：**

当前架构是 Go HTTP 服务器 + 嵌入式 Vue 网页。要把 UI 放进原生窗口，
需要引入 **桌面 GUI 框架**，这不是改几行代码的事：

| 方案 | 可行性 | 问题 |
|------|--------|------|
| **Wails v2**（推荐） | Go 原生 WebView 框架，可复用现有 Vue UI | 需要重构项目结构（wails.json, app 生命周期）、引入 Wails CLI 构建链、WebView2 运行时依赖（Win10/11 自带）。约 3-5 小时工作量 |
| **webview_go** | 轻量 WebView 封装 | 需要 CGO（Windows 上需安装 MinGW），交叉编译困难 |
| **Edge --app 模式** | 一行代码，无浏览器 chrome | 本质仍是浏览器进程，用户明确拒绝 |
| **Fyne / Walk** | 纯 Go 原生 GUI | 无法渲染 Vue，需完全重写 UI，不现实 |

**推荐路线：** Wails v2 迁移。现有 Vue 组件和 Go API 处理器都能复用，
但需要作为一个**独立里程碑**来做，不能在 E2E 测试期间同时改架构。

**前置条件：**
1. 安装 Wails CLI（`go install github.com/wailsapp/wails/v2/cmd/wails@latest`）
2. 运行 `wails doctor` 确认环境（需要 WebView2、Node.js）
3. 用 `wails init` 创建项目骨架，迁移现有代码
4. 测试构建 + 打包

### D. 多设备同时连接（#2 完整版）— 大改动

**为什么现在不能做：**
- SessionStore 只存一个 session，需要改成 List
- 设备列表 UI 需要重新设计（选择目标设备 → 传输）
- WS 连接管理需要从单连接变成连接池
- 传输服务需要按设备路由
- 估计 1-2 天工作量，且会大幅改变现有架构

建议 Phase 2 规划。

## 不做 🚫

### E. PC↔PC / 手机↔手机互传（#5）

**为什么不做：**
- 设计文档明确 Phase 1/2 范围 = Android ↔ Windows
- PC↔PC 需要 Go 增加"客户端模式"（中等改动）
- 手机↔手机需要 Flutter 运行 HTTP 服务器 + Android 后台保活（大改动，且 Android 限制多）
- 架构上可行（两通道设计不需要推翻），但属于 Phase 3+ 远期目标
